2. PHASE 1 — CONSTRAINT MAPPING & LIBERATION ZONES
───────────────────────────────────────────────────────────────────────────────

2.1 Immutable Constraints (physics, cannot circumvent)

    • Stepper motors require copper windings and permanent magnets (or iron
      cores). Cannot print copper conductors at genesis scale.
    • Sub-100 nm semiconductor lithography requires photolithography tools
      beyond additive printing capability. ICs remain an external input.
    • Hotend heater blocks require metal (aluminium, steel) for thermal mass
      and conductivity beyond what polymer can provide at 500 °C.
    • Thermistors and thermocouples require specific metal alloy compositions.
    • Belts require elastomeric tensile strength; TPU printed belts at
      generation 2 achieve ~90% of steel-core GT2 performance.

2.2 Practical Constraints (current knowledge, optimise within)

    • MGN12H linear rails are the best-in-class for X axis at current
      technology. Polymer alternatives achieve 85-90% capability.
    • Silver paste PCB traces reach ~50 mΩ/sq — adequate for signal and
      low-current power, not for high-current power distribution.
    • Raspberry Pi CM4 is the optimal SBC for Klipper + EVER node at current
      availability. RISC-V equivalent in generation 3+ target.

2.3 Assumed Constraints — CHALLENGED

    Convention Challenged           │ Result of Challenge
    ────────────────────────────────┼────────────────────────────────────────
    "Rectangular frame is standard" │ CHALLENGED → Spaceframe node + rod
                                    │ architecture adopted for generation 2.
                                    │ Gen 1 uses extrusion for bootstrap.
    "CoreXY is the best motion"     │ EVALUATED → CoreXY confirmed optimal
                                    │ for toolchanger mass; AWD CoreXY
                                    │ evaluated and REJECTED (complexity
                                    │ without sufficient benefit at this
                                    │ scale).
    "Linear rails everywhere"       │ CHALLENGED → X axis retains MGN12H
                                    │ (highest precision, highest load).
                                    │ Y axis: printed polymer V-slot guides
                                    │ on hardened rod (printable, 88%
                                    │ capability, replicable).
                                    │ Z axis: printed IGUS-style linear
                                    │ bushings on 8mm chrome rod.
    "Single extruder"               │ CHALLENGED → 5-position toolchanger
                                    │ adopted. Single extruder cannot
                                    │ fulfil universal material mandate.
    "Klipper or Marlin"             │ EVALUATED → Klipper confirmed.
                                    │ Real-time MCU + host processing split
                                    │ is architecturally superior. Marlin
                                    │ on single MCU cannot run aequchain
                                    │ node concurrently.
    "DLP needs projector"           │ CHALLENGED → MSLA (LCD mask +
                                    │ UV LED array) adopted. Achieves 47µm
                                    │ XY at 1/10 the cost. Sufficient for
                                    │ PCB-scale features.
    "Paste needs pneumatics"        │ CHALLENGED → Motor-driven auger
                                    │ pump adopted as primary. No compressor
                                    │ required. Pneumatic secondary for
                                    │ high-viscosity materials only.

2.4 Liberation Zones (design freedoms)

    • Frame can be designed for disassembly and flat-pack shipping.
    • The printer can be its own aequchain node — no separate server needed.
    • Tool 5 (laser) can double as PCB exposure tool, eliminating need for
      separate DLP unit in some PCB workflows.
    • All tool bodies are standardised — same kinematic coupling, same CAN
      bus connector, same mounting geometry. Any tool can occupy any dock.

3. PHASE 2 — FIRST-PRINCIPLES MULTIDIMENSIONAL CONCEPTION
───────────────────────────────────────────────────────────────────────────────

3.1 Dimensional Analysis Matrix

    FUNCTIONAL:
    The printer must deposit material with precision, in multiple material
    classes, across the full build volume, repeatably and reliably.
    → Requires: precise motion, thermal control, multi-tool capability,
      auto-calibration, failure detection.
    → Form factor: CoreXY gantry (best XY speed/precision), triple Z
      (auto-tramming), enclosed (thermal control), toolchanger (multi-material).

    EFFICIENCY:
    Minimise: energy per print, time per print, waste material, maintenance
    time, operator intervention.
    → Input shaping eliminates ringing → enables higher print speeds.
    → CAN bus reduces wiring → less failure modes, faster assembly.
    → MSLA PCB tool uses UV LED at 10W vs projector at 200W.
    → Triple Z with auto-tramming eliminates manual bed levelling time.
    → Pressure advance (Klipper) eliminates over/under-extrusion waste.

    ROBUSTNESS:
    Must operate 24/7 in community environments without specialist support.
    → Redundant thermal runaway protection (hardware + software).
    → Sensorless homing eliminates fragile limit switches.
    → All wiring uses locking connectors (no push-fit that vibrates loose).
    → ADXL345 resonance sensing for live print quality monitoring.
    → Camera-based AI failure detection (octoeverywhere or moonraker-timelapse).

    SCALABILITY:
    Single unit must scale to farm without architectural change.
    → Each unit is identical. Network auto-discovers via aequchain.
    → Farm management via Moonraker + Farm-PI or OctoFarm.
    → Specialised units are same hardware, different tool complement.
    → No central controller required — fully decentralised.

    MAINTAINABILITY:
    Must be repairable by a community member with basic training.
    → All electrical connections: JST-XH and Molex Micro-Fit. No soldering.
    → All printed parts: files on-board SD card and aequchain network.
    → Tool-less access to electronics bay (quarter-turn fasteners).
    → Colour-coded wiring harness.
    → Self-diagnostic Klipper macros for fault identification.

    INNOVATION:
    Transcend convention where beneficial.
    → Integrated aequchain EVER node: printer is simultaneously production
      tool and economic node. No other open-source printer does this.
    → Laser tool doubles as PCB exposure: eliminates dedicated DLP unit
      for simple single-layer PCBs.
    → Auger paste extruder enables continuous paste extrusion without
      syringe refill interruptions.
    → Printed polymer V-slot guides on Y axis: reduces cost and increases
      self-replication index.
    → Flat-pack frame design: ships at 1/6 assembled volume.

    ELEGANCE:
    Maximum purpose achievement / minimum complexity.
    → One CAN bus harness replaces 40+ individual motor/sensor wires.
    → Three-point kinematic bed eliminates over-constraint (no 4/6 point).
    → Single standardised tool coupling replaces bespoke per-tool interface.
    → One firmware (Klipper) manages all five tools without sub-firmware.

3.2 Concept Generation & Selection

    Concept A: Industrial Cartesian (simple, slow, less versatile)
    → REJECTED: Insufficient speed and versatility for community production.

    Concept B: Delta (fast, limited for paste, complex calibration)
    → REJECTED: Paste extrusion on delta is geometrically problematic.
      Calibration complexity too high for community operation.

    Concept C: CoreXY with standard single extruder
    → REJECTED: Cannot fulfil universal material mandate.

    Concept D: CoreXY toolchanger (E3D-style)
    → PARTIALLY ACCEPTED: Motion system selected. Tool coupling design
      improved to Jubilee-class kinematic coupling.

    Concept E: SCARA robot (fast, large envelope, complex)
    → REJECTED: Tool changing extremely difficult. Not self-replicating.

    Concept F: Cable-driven parallel robot (lightweight, scalable)
    → DEFERRED: Promising for generation 3 scaled variant. Not suitable
      for precision PCB printing at genesis.

    SELECTED ARCHITECTURE:
    CoreXY motion + Triple Z + 5-position Jubilee-class toolchanger +
    enclosed heated chamber + dual PSU + Klipper/CM4 + CAN bus +
    aequchain EVER node integration.

    This synthesis scores highest across all 7 OPTIBEST dimensions.

4. PHASE 3 — SEQUENTIAL ITERATIVE DESIGN DECISIONS
───────────────────────────────────────────────────────────────────────────────

Each decision below was made, questioned, iterated, and confirmed.

4.1 Frame Architecture

    DECISION 1: Aluminium 2020 extrusion + printed nodes (bootstrap).

    Iteration 1: Standard aluminium extrusion only.
    Question: Is this optimal for self-replication? NO — all metal, none
              printable.
    Iteration 2: Add printed corner brackets. Reduces metal but frame still
              fully dependent on extrusion lengths.
    Question: Can we design for printed replacement of extrusion? YES.
    Iteration 3: Design frame as NODE + ROD system. Printed polyhedra nodes
              at each corner. Standard steel/aluminium tube rods between them
              (replaceable with PETG-CF printed tubes in gen 2).
    Question: Does node+rod achieve same rigidity as extrusion? Need check.
    Analysis: 2020 aluminium extrusion: ~2000 N/mm² bending stiffness.
              30mm OD × 2mm wall PETG-CF tube: ~800 N/mm² — adequate for
              non-gantry members. Gantry cross members retain aluminium.
    Iteration 4 (FINAL): Hybrid — aluminium 2040 extrusion for X and Y
              gantry rails (highest load path). Aluminium 2020 for vertical
              frame members (bootstrap). Printed PETG-CF nodes at all
              intersections. Gen 2 replaces 2020 verticals with printed
              PETG-CF tubes. Gen 3 replaces gantry rails.

    CONFIRMED: Cannot improve further. Hybrid adopted.

4.2 Motion System

    DECISION 2: CoreXY with 9mm GT2 belts, MGN12H on X, printed V-slot on Y.

    Iteration 1: CoreXY with MGN12H on both axes.
    Question: Is MGN12H on Y essential? No — Y axis has lower precision
              requirement (gantry only moves in Y, tool moves in X).
    Iteration 2: Y axis uses printed polymer V-slot guides on 8mm hardened rod.
    Question: Does 88% stiffness on Y impact print quality? Analyse:
              Y axis acceleration: 3000 mm/s². Force on 500g gantry: 1.5N.
              Printed guide deflection at 1.5N: ~0.02mm. Input shaping
              compensates. Print quality impact: negligible.
    Question: Is 9mm belt optimal? GT2 6mm standard. 9mm wider belt:
              +125% tensile strength, same pitch precision. For toolchanger
              (heavier carriage), 9mm is clearly superior.
    Iteration 3 (FINAL): MGN12H on X (precision critical, toolchanger loads).
              Printed IGUS-style V-slot guides on hardened 10mm rod on Y.
              9mm GT2 belts on both axes. Dual motors per axis not needed
              (AWD evaluated: adds 2 motors, +$60, benefit marginal at this
              speed). Single motor per axis confirmed.

    Z-AXIS DECISION 3: Triple lead screw, auto-tramming.

    Iteration 1: Single lead screw centre.
    Question: Risk of Z tilt? YES — inevitable on single lead screw.
    Iteration 2: Dual lead screws + sync belt.
    Question: Can auto-tram? Partially — dual axis allows X-tilt correction.
    Iteration 3: Triple lead screws at 120° from build plate centre.
    Question: Full auto-tramming possible? YES — Klipper z_tilt_adjust.
              Corrects all tilt axes. Enables perfect first layers automatically.
    Question: More complex? Yes. Worth it? YES — eliminates manual levelling
              entirely, which is the single most common printer maintenance
              task. Operator time saved: 15-30 min per week.
    Iteration 4 (FINAL): Triple T8 lead screws (8mm pitch per rev, 2mm lead)
              at 120°, each with independent NEMA17 motor. Klipper
              z_tilt_adjust + mesh bed levelling. Automatic every print.

    CONFIRMED: Cannot improve further. Triple Z adopted.

4.3 Toolchanger Design

    DECISION 4: Jubilee-class kinematic 3-point spring coupling, 5 docks.

    Iteration 1: E3D toolchanger style (magnetic coupling).
    Question: Repeatability adequate? ±5µm typical. Adequate for 200µm PCB
              traces? Marginal. Improve.
    Iteration 2: Jubilee-style kinematic coupling (3 cantilevered balls into
              3 vee grooves). Repeatability: <2µm theoretical.
    Question: Can we print the coupling components? Balls: hardened steel
              (6mm drill bit tip, commodity). Vee grooves: printed PETG-CF
              insert hardened with CA glue + sand = adequate.
    Iteration 3: Spring tension. Jubilee uses 5N spring. For 5-tool heavy
              complement: increase to 8N. Printed spring carriers, metal
              springs (commodity, 5p each).
    Question: Is 5 tool positions optimal? Evaluate: Tool 1 (HT FFF),
              Tool 2 (standard FFF), Tool 3 (paste), Tool 4 (MSLA),
              Tool 5 (laser). Could merge any? Tool 4 and 5 have some
              overlap (PCB exposure) but different capabilities. Keep 5.
              Could add 6th? Reserve dock slot in frame — 6th tool future.
    Iteration 4 (FINAL): 5-dock Jubilee-class toolchanger. 3-ball kinematic
              coupling per tool. 8N spring engagement. All tool bodies same
              external geometry. CAN bus + 4-pin power pogo pin connector
              on each dock. Tool weight limit: 500g each.

    CONFIRMED: Cannot improve further. Jubilee-class adopted.

4.4 Hotend Architecture (Tool 1)

    DECISION 5: All-metal hotend, 500°C, hardened steel nozzle.

    Iteration 1: V6 clone all-metal.
    Question: Adequate for 500°C PEEK? V6 rated to 450°C max. NO.
    Iteration 2: E3D Revo or equivalent high-temp.
    Question: Revo is proprietary. Need open equivalent.
    Iteration 3: Dragon/Rapido-style all-metal with titanium heat break +
              bimetallic heater block (steel body, copper core insert).
              Max 500°C with TYPE-K thermocouple (vs NTC3950 limit 300°C).
    Question: Is titanium heat break printable? NO — titanium machined.
              However: 1 heat break per printer, commodity sourced at $3.
    Question: Nozzle material? Hardened steel (0.4mm standard). For abrasive
              filaments: hardened steel survives. For softer materials:
              brass insert available. Both are commodity.
    Iteration 4 (FINAL): Titanium heat break + bimetallic block + hardened
              steel nozzle + TYPE-K thermocouple + 80W ceramic heater
              cartridge. Heater rated 24V, 80W. Hotend fan: 5015 blower
              (always on when hotend > 50°C). Part cooling: 5015 blower
              with PETG-CF printed duct (printable, replaceable).

    CONFIRMED: Optimal. Cannot improve further.

4.5 Paste Extruder (Tool 3)

    DECISION 6: Dual-mode — motor-driven auger primary, pneumatic secondary.

    Iteration 1: Syringe-only pneumatic.
    Question: Requires compressor? YES — significant added BOM cost and
              noise. Can avoid?
    Iteration 2: Motor-driven syringe (lead screw + stepper).
    Question: Adequate for all pastes? For low-viscosity (Ag paste, bio-ink):
              YES. For high-viscosity (silicone, concrete at small scale): NO.
    Iteration 3: Motor-driven auger (archimedes screw).
    Question: Benefits over syringe? Continuous extrusion (no refill stops),
              handles higher viscosity, printable auger body and screw.
              Question: Can print auger in PETG-CF? YES for most pastes.
              For concrete: print in PETG-CF with replaceable wear insert.
    Question: Should we eliminate pneumatic entirely? Evaluate: Genesis scale
              concrete printing is not a priority for Tool 3. Pneumatic is
              reserved for Tool 3B (expansion module). Base design: auger only.
    Iteration 4 (FINAL): Motor-driven auger extruder. NEMA17 motor (same as
              motion system — standardised). Printed PETG-CF barrel. Stainless
              steel auger screw (commodity, 8mm diameter, sourced from
              pharmacy mixing equipment or printed metal-fill + sintered).
              Interchangeable nozzle tips: 0.4mm (Ag paste), 0.8mm (solder),
              1.5mm (bio-ink), 3mm (food-grade gel). Tip change: tool-less
              twist-lock. Temperature-controlled barrel (optional, for
              temperature-sensitive materials).

    CONFIRMED: Auger primary adopted. Cannot improve further.

4.6 MSLA PCB Tool (Tool 4)

    DECISION 7: MSLA with 405nm UV LED array + monochrome LCD mask.

    Iteration 1: DLP projector.
    Question: Cost? $200-800. Fragile lamp/DMD. Proprietary. Not EFE.
    Iteration 2: UV LED + LCD mask (MSLA approach).
    Question: Resolution? Standard 5.5" monochrome LCD: 2560×1440 at 68µm.
              For 200µm traces: MORE than adequate. For 100µm experimental:
              adequate with 2× optical reduction.
    Question: Can we print the LCD housing? YES — PETG-CF housing, M3
              standoffs. LCD is commodity (display panel from phone/tablet
              supply chain).
    Question: Build area for PCBs? At 68µm pixel pitch: 5.5" LCD gives
              ~174×98mm exposure area. Sufficient for controller boards and
              most toolhead PCBs. Larger PCBs: tile exposure (stitch).
    Iteration 3 (FINAL): 5.5" monochrome LCD (2560×1440, 68µm pixel pitch)
              + 405nm UV LED array (4×4 matrix, 100mW each, 1.6W total) +
              Raspberry Pi Zero W (WiFi-controlled, runs ChiTuBox or NanoDLP
              server for exposure control) + printed PETG-CF housing.
              Cure time: 2-8 seconds per layer (dielectric substrate).
              Z movement: provided by printer Z axis (tool is static exposure
              unit — no own Z drive needed). Build area: 174×98mm.

    CONFIRMED: MSLA adopted. Cannot improve further.

4.7 Laser Tool (Tool 5)

    DECISION 8: 5W 450nm diode laser (galvo optional in gen 2).

    Iteration 1: Diode laser with fixed optics.
    Question: 5W adequate for: engraving PETG → YES (2-3 mm/s).
              Sintering metal-fill filament surface → YES partially.
              PCB exposure (alternative to MSLA) → YES for simple masks.
              Cutting 3mm PETG → marginal (requires slow pass).
    Question: Safety? Laser class 4. Enclosed chamber provides safety.
              Interlock on door: hardware disconnect. Goggles required if
              door open during operation.
    Iteration 2: 10W laser.
    Question: Is 10W needed? For genesis role: 5W handles all intended tasks.
              10W improves throughput but adds cost and safety requirements.
              Retain 5W. 10W upgrade documented as gen 2 enhancement.
    Iteration 3 (FINAL): 5W 450nm diode laser module (TTL modulated 0-5V
              control, matches Klipper laser control macro). Printed PETG-CF
              mount with adjustable focus (6mm thread focal length adjust).
              Air assist nozzle (printed) connected to optional small pump.
              Safety interlock: magnetic door sensor → hardware laser kill.

    CONFIRMED: 5W diode adopted. Cannot improve further.

4.8 Electronics Architecture

    DECISION 9: Raspberry Pi CM4 (host) + STM32H723 MCU (real-time) +
                TMC5160 on X/Y/Z, TMC2209 on extruders + CAN bus toolheads.

    Iteration 1: Standard SKR Pro + Klipper.
    Question: Can run aequchain EVER node? NO — insufficient compute.
    Iteration 2: Raspberry Pi 4B + STM32 MCU.
    Question: RPi 4B adequate? YES for Klipper + EVER node. Prefer CM4
              for compactness and reliability (no SD card main storage —
              use eMMC on CM4).
    Iteration 3: CM4 + Octopus Pro equivalent MCU.
    Question: STM32H723 superior to STM32F4xx? YES — 550 MHz vs 168 MHz,
              hardware FPU, better for input shaping computation.
    Question: TMC5160 vs TMC2209 vs TMC2240?
              TMC5160: SPI, 3A continuous, best for X/Y/Z (high current,
              full diagnostic). Premium.
              TMC2209: UART, 2A, adequate for extruder motors (lower load).
              TMC2240: SPI, 3A, adds temperature monitoring — consider for
              gen 2 upgrade.
              DECISION: TMC5160 for all motion axes, TMC2209 for extruders.
    Iteration 4: CAN bus toolheads.
    Question: Each tool needs: stepper driver, heater control, thermistor,
              fan control, possibly LED. Is CAN bus worth the complexity?
              YES — replaces 40-60 individual wires with 4-wire CAN harness.
              Reliability dramatically improved. Wiring time: 2 hours → 20
              minutes. Use: Huvud or EBB36 toolhead CAN boards (open-source).
    Iteration 5 (FINAL): CM4 (4GB, 32GB eMMC) + STM32H723-based mainboard
              (Spider v3 or Octopus Pro open equivalent) + TMC5160 on
              XY + 3×Z + TMC2209 on E0-E4 (5 extruder slots) +
              CAN bus at 1Mbit/s on each toolhead + USB-C power delivery
              monitoring + Ethernet primary, WiFi backup.

    CONFIRMED: Cannot improve further. Architecture locked.

4.9 Power Architecture

    DECISION 10: Dual PSU — 24V/600W primary, 48V/600W chamber/high-temp.

    Iteration 1: Single 24V/600W PSU.
    Question: Adequate for heated chamber? 500W chamber + 80W hotend +
              motion motors 100W + electronics 50W = 730W. NO.
    Iteration 2: Single 24V/1500W PSU.
    Question: Chamber at 24V requires 62.5A for 1500W. Cable gauge
              requirements become impractical. Better to use 48V.
    Iteration 3: Dual: 24V/600W (electronics/motion) + 48V/600W (chamber/HV).
    Question: Are AC power connections manageable? YES — two standard IEC C13
              inputs. Alternatively: one 24V/600W + one 48V/600W, both from
              split-phase or standard 230V single phase.
    Question: Solar/off-grid input? Add 12-60V MPPT input → DC-DC boost to
              48V bus. Allows solar charging with no inverter needed.
    Iteration 4 (FINAL):
              PSU 1: Mean Well RSP-600-24 (24V, 25A, 600W) → motion,
                     electronics, extruder heaters, fans.
              PSU 2: Mean Well RSP-600-48 (48V, 12.5A, 600W) → chamber

              MPPT Input: 15-60V DC input → 48V bus for solar/wind.

    CONFIRMED: Cannot improve further. Dual PSU adopted.

4.10 Enclosure & Thermal Design

    DECISION 11: Double-wall PETG panel enclosure with PTC + recirculation.

    Iteration 1: Single PETG panel enclosure.
    Question: Thermal efficiency? Poor — PETG is 0.25 W/(m·K) thermal
              conductivity. Single 3mm panel loses too much heat for PEEK.
    Iteration 2: Dual-wall with air gap (insulating layer).
    Question: Can print dual-wall panels? YES in sections. Gap: 20mm.
              Effective insulation: ~0.04 W/(m·K). Chamber heating time
              to 70°C: ~8 minutes (vs 15 minutes single wall). 47% faster.
    Question: What heats the chamber? PTC ceramic heater, 500W, 48V.
              Recirculation fan: 80mm brushless. Temperature control: Klipper
              heater PID.
    Question: Electronics isolation? Electronics bay external to heated
              chamber. Active filtered air intake for electronics (HEPA).
    Iteration 3 (FINAL): Double-wall PETG panels (printed, 3mm inner + 20mm
              gap + 3mm outer, corners joined with printed PETG clips).
              PTC ceramic heater 500W/48V with printed PETG housing (ceramic
              inserts provide fire safety). 80mm recirculation fan. Door:
              printed PETG frame + borosilicate glass panel (toolless removal:
              quarter-turn magnetic latches). Top panel: printed PETG hinged
              access. HEPA filter on electronics bay air intake.

    CONFIRMED: Cannot improve further.

4.11 Flat-Pack Transport Design

    DECISION 12: Designed for disassembly into 600×600×200mm package.

    Iteration 1: Standard assembled printer.
    Question: Transport difficulty? Significant. Fragile gantry. High volume.
    Iteration 2: Removable gantry.
    Question: How detach gantry? Printed clip system on Z uprights. 4 clips.
              Gantry lifts off. Nest gantry inside frame footprint.
    Question: Can electronics separate easily? YES — CM4 module in printed
              electronics bay. JST connectors throughout. 5-minute disconnect.
    Iteration 3 (FINAL): Flat-pack sequence:
              1. Remove tools from docks (5 × ~30 sec = 2.5 min)
              2. Disconnect CAN bus harness (1 clip, 1 min)
              3. Clip-off gantry (4 clips, 1 min)
              4. Remove electronics bay panel (quarter-turn, 30 sec)
              5. Release bed assembly (3 leadscrew nuts, 2 min)
              6. Frame: knock apart printed nodes + rod tubes (5 min)
              Total: ~12 minutes to flat-pack
              Total: ~25 minutes to reassemble
              Package: 600×600×200mm → fits airline cargo, motorcycle
              sidecar, bicycle trailer, standard shipping box.

    CONFIRMED: Cannot improve further.

4.12 aequchain EVER Node Integration

    DECISION 13: Klipper macros log print completions to EVER node via REST.

    Iteration 1: Separate computer for EVER node.
    Question: Necessary? NO — CM4 has adequate compute for both Klipper and
              EVER node client.
    Iteration 2: EVER node software on CM4.
    Question: Resource conflict with Klipper? Klipper uses <200MB RAM.
              CM4 4GB has 3.8GB headroom. EVER node (Julia process):
              ~400MB. No conflict.
    Iteration 3: Integration depth. Options:
              (a) Simple: Klipper macro calls EVER REST API at print end.
              (b) Deep: Klipper streams print metrics to EVER in real-time.
              (c) Moderate: Batch epoch sync every hour.
    Question: Which best serves EFE purpose? (b) is overkill. (a) is
              insufficient for production tracking. (c) SELECTED:
              Hourly epoch sync of: print hours, material used, jobs
              completed, estimated output value. Klipper MQTT broker
              → EVER node listener → aequchain epoch submission.
    Iteration 4 (FINAL): Klipper runs Moonraker (REST API). EVER node client
              polls Moonraker every 60 minutes. Extracts: print_duration,
              filament_used (g), estimated_value (configurable per-material
              rate card from aequchain network). Submits signed production
              record to aequchain testnet/mainnet epoch. All code: MIT
              licensed, in /home/ever/ on CM4. Auto-starts at boot.

    CONFIRMED: Cannot improve further.

5.1 Multi-Attempt Enhancement (4 attempts minimum)

    ATTEMPT 1: Replace CoreXY with COREXY-AWD (4 motors on XY)
    Analysis: AWD adds 2 motors ($30), adds complexity, reduces belt failure
              risk marginally. At 300×300mm build area and toolchanger mass
              of 300g, standard CoreXY belt tension is adequate. AWD yields
              no measurable quality improvement at this scale.
    Result: REJECTED. No enhancement. CoreXY confirmed.

    ATTEMPT 2: Add 6th tool dock
    Analysis: 6th dock adds 12% to width. Adds one more cable. 5 tools
              cover: HT FFF, standard FFF, paste, DLP, laser. What would
              6th tool be? Milling/routing. Valid use case, but adds spindle
              motor, chip extraction, vibration management. Architecture
              already provides 6th dock slot physically. Leave physically
              present, firmware-disabled. Future upgrade documented.
    Result: 6th dock SLOT retained in frame (no cost, no complexity).
             6th tool deferred to gen 2 upgrade. Enhancement: MINOR structural
             accommodation added (dock slot machined into printed dock plate).

    ATTEMPT 3: Eliminate CM4 in favour of single powerful MCU
    Analysis: All-MCU approach (RP2040 cluster or STM32H7 only) is possible
              but eliminates Linux environment needed for EVER node. Cannot
              run aequchain Julia client without Linux host. Purpose failure.
    Result: REJECTED. CM4 + MCU split confirmed as optimal.

    ATTEMPT 4: Heated bed to 200°C instead of 120°C
    Analysis: 200°C bed needed for: some ceramics, metal sinter preparation.
              Current design: PEI/PEX spring steel rated to 120°C. 200°C
              requires silicone bed surface + aluminium or steel bed plate +
              high-wattage heater. Adds $40 to BOM. Required for: sintered
              metal-fill (strong use case for self-replication). Include.
    Result: ENHANCEMENT ACCEPTED. Bed heater upgraded to 750W/48V silicon
              pad on 5mm aluminium plate. Rated 200°C. PEI surface replaced
              with G10/FR4 surface at high temp. PEI sheet retained for ≤120°C
              operation (magnetically swappable surface).

    ATTEMPT 5: Replace PETG-CF panel enclosure with polycarbonate
    Analysis: PC panels are optically clear (allows visual monitoring of full
              chamber without opening door). PC rated to 120°C continuously
              (chamber max 70°C, margin adequate). PC is printable (this
              printer prints PC). PC panels: more expensive to source at
              bootstrap but printable at gen 2. At genesis: purchase PC
              sheet ($25 total). Gen 2: print replacement panels.
    Result: ENHANCEMENT ACCEPTED. Side panels: 3mm PC sheet at bootstrap,
              printable PETG-CF from gen 2. Door panel: 3mm borosilicate
              (laser/UV safety) retained at all generations.

10. MOTION SYSTEM
───────────────────────────────────────────────────────────────────────────────

10.1 CoreXY Kinematics

    Two motors (X, Y) control motion in both axes via crossed belt geometry.
    Tool carriage is the only moving mass in XY — both motors are stationary.

    Belt routing (standard CoreXY):
    Motor A (rear left) ──▶ idler (front left) ──▶ idler (front right)
                          ──▶ carriage left attachment
    Motor B (rear right) ──▶ idler (front right) ──▶ idler (front left)
                           ──▶ carriage right attachment

    Belt: GT2 9mm width, 3mm² steel core, 2mm pitch.
    Idler: F695 flanged bearing, 5mm ID (printed carrier, metal bearing).
    Tension: 40N per belt (measured with Klipper resonance frequency method).

10.2 X Axis

    Rail: MGN12H × 2 (top and bottom of X gantry beam)
    Carriage: MGN12H block (standard) + printed PETG-CF adapter to toolchanger
    X motor: NEMA17 48mm (1.8°, 42 N·cm holding torque, socketed connector)
    X endstop: Sensorless (TMC5160 stall detection, no physical switch needed)

10.3 Y Axis

    Guide: 2× 10mm chrome-steel rod (500mm length each), mounted in printed
           IGUS-style dry-lube V-slot blocks.
    V-slot blocks: Printed PETG + 20% PTFE filament (self-lubricating).
                   Replace when worn (~6 months typical). Reprint in 15 min.
    Y motor: NEMA17 48mm (same as X — fully standardised)
    Y endstop: Sensorless (TMC5160)

10.4 Z Axis (Triple Lead Screw)

    3× T8 lead screw (2mm lead, 8mm/rev), 420mm length
    3× NEMA17 48mm motors (same — fully standardised)
    3× Printed PETG-CF anti-backlash nut blocks (spring-loaded)
    3× F608 flanged bearings at top (printed carrier)
    3× 8mm chrome rod guide per screw (printed IGUS bush)
    Z endstop: Inductive probe (Beacon or CR Touch — open-source designs)
               Mesh compensation: 49-point (7×7) mesh via Klipper.
    Auto-tramming: Klipper Z_TILT_ADJUST macro. Runs at start of every print.

10.5 Input Shaping

    ADXL345 accelerometer: mounted on toolhead via printed PETG-CF clip.
    Klipper SHAPER_CALIBRATE runs automatically after Z homing.
    Typical resonance: X axis 45-55 Hz, Y axis 35-45 Hz.
    Shaper type: MZV or EI (selected automatically).
    Effective max acceleration after shaping: 8000-12000 mm/s².

10.6 Motor Standardisation

    All motors: NEMA17 48mm, 1.8° step, 1.7A rated, 4-wire, JST-XH 2.54mm
    connector. Identical across XY, Z1, Z2, Z3, and extruder positions.
    Total motor count: 5 motion (XY + 3Z) + 5 extruder = 10 motors.
    Spare: 2 motors (12 total in BOM). Cost reduction through bulk.

11.1 Kinematic Coupling Design

    Coupling: 3-point ball-into-vee. Repeatability: <2µm.
    Balls: 6mm hardened steel (from commodity drill bits or bearing balls)
    Vee grooves: Printed PETG-CF V-channel inserts, coated with CA + abrasive
                 for hardness. Replace every 6 months or when repeatability
                 degrades beyond 5µm (checked via calibration print).
    Spring engagement: 8N compression spring (2× per tool). Loaded on dock.
    Passive coupling: Tool picked up by carriage pushing against dock spring.
                      No powered latch mechanism (pure passive = no failure mode).

11.2 Tool Dock Design

    5 docks mounted on rear frame, angled 15° for carriage access.
    Dock plate: Printed PETG-CF (replaceable). M5 bolts to frame.
    Each dock:
    • Spring-loaded tool support plate
    • 4-wire pogo-pin CAN bus connector (2× GND, CAN-H, CAN-L)
    • 2-wire pogo-pin 24V power (for heater standby)
    • 2-wire pogo-pin 48V power (for PEEK hotend standby pre-heat)
    • LED status indicator (green=ready, red=fault, blue=not selected)
    • QR code label (links to tool calibration procedure)

11.3 Tool-Change Procedure (automated, Klipper macro)

    1. Move carriage to current tool dock (at speed 200mm/s)
    2. Lower carriage into dock (Z move)
    3. Release coupling (carriage moves backward, spring re-engages dock)
    4. Wait: 50ms stabilisation
    5. Move carriage to target tool dock
    6. Pick up tool (carriage pushes into coupling, spring compresses)
    7. Validate: Klipper reads tool ID via CAN bus address confirmation
    8. Run tool-specific startup macro (set temperature, home if needed)
    Tool change time: ~8 seconds typical.

11.4 Tool Body Standard

    All 5 tools share identical external geometry on the coupling face:
    • 60mm W × 40mm D coupling plate
    • 3× 6mm ball seats at standard positions
    • 4-pin pogo connector at standard position
    • 2-pin + 2-pin power pogo at standard positions
    • M3 mounting holes for tool-specific components above coupling plate
    Tool body frame: Printed PETG-CF (replaced if damaged, ~30 min print).

12. BED & BUILD SURFACE
───────────────────────────────────────────────────────────────────────────────

12.1 Bed Construction

    Heated plate: 5mm aluminium tooling plate, 320×320mm, flatness ±0.05mm.
    Heater: 750W/48V silicon heater pad, bonded with high-temp adhesive.
    Thermal fuse: 230°C thermal cutoff, wired in series (hardware safety).
    Temperature sensor: 100K NTC thermistor (embedded in Al plate).
    Magnetic base: 3mm steel sheet bonded to aluminium, for spring-steel PEI.

12.2 Build Surfaces (swappable, magnetic)

    Surface A: PEI-coated spring-steel sheet (standard, ≤120°C, all standard
               filaments). Remove: flex and pop. Install: align magnetically.
    Surface B: G10/FR4 sheet (high-temp, ≤200°C, PEEK, PC, Nylon).
               Also: PCB substrate when printing bare circuit boards.
    Surface C: Borosilicate glass (resin/DLP, adhesive-free release).

12.3 Bed Mounting (Kinematic 3-point)

    3 mounting points at 120° from centre. Each point: M5 ball + printed
    PETG V-seat. Spring-loaded. No overconstrain. Auto-levelling compensates
    remaining variation.
    Bed repeatability after auto-tram: ±0.01mm across full surface.

12.4 Z Offset Calibration

    Beacon probe (open-source, eddy-current, no contact, 1µm resolution)
    mounted on Tool 1 body. Scans bed mesh before each print.
    Alternatively: CR Touch (open mechanism, widely available).
    Both compatible. Klipper handles both via config switch.

13. ENCLOSURE & THERMAL MANAGEMENT
───────────────────────────────────────────────────────────────────────────────

13.1 Panel Construction

    Side panels (4×): 3mm polycarbonate sheet, 500×700mm each.
                      Frame: printed PETG-CF border with snap-fit clips.
                      Gen 2: replace with printed PETG-CF double-wall panels.

    Bottom panel:     3mm PC sheet + 10mm printed PETG-CF feet (4×, anti-

13.2 Chamber Heating

    PTC heater: 500W / 48V, self-limiting (PTC = self-regulating above Curie
                temperature). No thermal runaway risk from heater itself.
    Housing: Printed PETG-CF with ceramic insert pockets (PTC elements slot in).
    Recirculation fan: 80mm axial fan, 12V, 45 CFM. Circulates air over PTC,
                       distributes heat evenly through chamber.
    Control: Klipper heater PID (separate PID from hotend). Sensor: AHT20.
    Heat-up time to 70°C: ~8 minutes (from 20°C ambient).
    Thermal stratification: ±3°C across chamber (with recirculation).

13.3 Electronics Bay

    Isolated from heated chamber. Rear of machine.
    Active cooling: 120mm fan, filtered intake (HEPA cartridge).
    Temperature target: ≤45°C at all times.
    Heat sources: CM4 (~5W), MCU board (~15W), stepper drivers (~20W total).
    Cooling capacity of 120mm fan at 50 CFM: more than adequate.

13.4 Air Management

    Intake: Bottom-front (filtered). Hot air rises through chamber.
    Exhaust: Top-rear (through HEPA/activated carbon filter).
    Filter housing: Printed PETG-CF. Filter cartridge: replaceable.
    Printing ABS/ASA: Filter active carbon removes styrene.
    Printing PEEK: VOC sensor (optional, EVER node logs air quality data).

23. PCB PRINTING CAPABILITY
───────────────────────────────────────────────────────────────────────────────

23.1 Complete PCB Workflow (in-house, no fab required)

    STEP 1 ──────────────────────────────────────────────────────────────────────
    SUBSTRATE PRINT (Tool 4 - MSLA)
    Material: Ceramic-loaded photopolymer resin (FR4-equivalent).
    Load Surface C (borosilicate) onto bed.
    Slice in NanoDLP: 0.1mm layers, 4 second cure per layer.
    Print substrate layers: 8 layers = 0.8mm thick substrate.
    Remove. Post-cure: 405nm flood lamp, 10 minutes.
    Result: Rigid, flat, 174×98mm substrate (or smaller as needed).

    STEP 2 ──────────────────────────────────────────────────────────────────────
    DIELECTRIC LAYER (Tool 4 or Tool 3)
    Print thin dielectric paste layer over substrate surface.
    Material: Dielectric UV paste (Tool 4) or dielectric thermal paste (Tool 3).
    Cure: 405nm, 2 seconds.
    Result: 0.1mm dielectric isolation layer.

    STEP 3 ──────────────────────────────────────────────────────────────────────
    CONDUCTIVE TRACE LAYER (Tool 3 - Paste Auger)
    Paste: Silver paste (Ag 70% particle content, IPA carrier).
    Nozzle: 0.4mm tip.
    Speed: 8 mm/s.
    Pressure advance: 0 (paste incompressible).
    Z offset: 0.15mm above substrate (paste squishes to 0.1mm trace height).
    Cure: Heated bed to 150°C, hold 30 minutes.
    Result: Ag traces, ~50 mΩ/sq, 0.2mm minimum width.

    STEP 4 ──────────────────────────────────────────────────────────────────────
    SOLDER PASTE DEPOSITION (Tool 3 - Paste Auger, swap nozzle to 0.8mm)
    Paste: SAC305 solder paste (commodity, ~$15/syringe).
    Deposit only on SMD component pads.
    Accuracy: ±0.1mm (adequate for 0402 and larger packages).

    STEP 5 ──────────────────────────────────────────────────────────────────────
    COMPONENT PLACEMENT (Manual or semi-auto)
    Place SMD components using tweezers (or printed vacuum pick tool).
    Tool 5 (laser) at low power can thermally mark placement targets.

    STEP 6 ──────────────────────────────────────────────────────────────────────
    REFLOW (Heated bed)
    Bed to 250°C (SAC305 liquidus: 218°C). Hold 60 seconds. Ramp down.
    All placed components reflow soldered.
    Visual inspection: Camera 1 captures before/after images.

    STEP 7 ──────────────────────────────────────────────────────────────────────
    CONFORMAL COAT (Tool 3 or manual)
    Apply UV conformal coat (thin photopolymer) over completed board.
    Cure with Tool 4 (MSLA array, flood exposure 30 seconds).
    Result: Moisture + vibration protected, professional finish.

27.1 Filament Self-Production (Target: Phase 2)

    EQUIPMENT REQUIRED (all printable):
    • Plastic shredder (printed PETG-CF housing, hardened steel blades,
      NEMA23 motor, commodity)
    • Granulator (printed housing, cutting disc, motor)
    • Filament extruder (printed barrel, auger, die, winding spool motor)
      Reference: Filastruder-style design, open-source, CC0.

    PROCESS:
    1. Collect waste plastic (PLA prints, PETG offcuts, recycled containers)
    2. Shred → granulate → dry (50°C / 4 hours)
    3. Extrude at 220°C through 1.75mm die
    4. Diameter control: laser caliper (print housing, commodity laser sensor)
    5. Wind onto printed spool

    COST REDUCTION:
    Purchased filament: ~$25/kg (PETG)
    Recycled filament: ~$2/kg (pellets) + energy
    Circular waste stream: ~$0.50/kg equivalent

27.2 Silver Paste Self-Production (Target: Phase 3)

    Silver nanoparticle paste:
    • Silver nitrate reduction with ascorbic acid (vitamin C)
    • Produces Ag nanoparticles in solution
    • Concentrate + mix with binder (ethyl cellulose in terpineol)
    • Result: printable Ag paste, ~$8/g vs ~$30/g commercial
    Equipment: Printed PETG-CF reaction vessel, stirrer motor (commodity)
    All chemistry: non-hazardous, open-source recipes available.

27.3 EFE Material Hierarchy Applied

    ✓ Preferred: Bio-PLA, bio-TPU, recycled PETG, recycled ABS
    ✓ Acceptable: Nylon, PC (with recycling pathway documented)
    ✓ Acceptable: Silver paste (small quantities, essential for PCB)
    ✗ Avoided: Virgin fossil-fuel PLA if bio-equivalent available
    ✗ Prohibited: Filaments with undisclosed additives, conflict-sourced metals

35. aequchain EVER NODE ARCHITECTURE
───────────────────────────────────────────────────────────────────────────────

35.1 Node Architecture

    Each Genesis Printer = 1 aequchain EVER production node.
    Node identity: ED25519 key pair generated at first boot.
    Public key: registered on aequchain EVER network (local testnet or
                production mainnet when audited).
    Node type: PRODUCTION_NODE (not a consensus node — lightweight).

35.2 Value Calculation

    Production value per epoch (hourly):
    ┌──────────────────────────────────────────────────────────────────
    │ material_used_g × material_rate[material_id]
    │ + print_hours × service_rate
    │ - input_cost (filament purchased from network treasury)
    │ = net_production_value
    └──────────────────────────────────────────────────────────────────

    Rate card: Set by aequchain EVER network governance each epoch.
    All printers in network use same rate card. No individual pricing.

35.3 EVER Ledger Effect

    Net production value added to Network Treasury.
    Network Treasury / Total Members = each member's equal balance.
    Adding more printers → more treasury → higher per-member balance.
    No printer "earns more" for its owner. All produce for all.

35.4 Free Living Index Tracking

    aequchain EVER tracks: which production chains are internalised.
    Each internalised chain: removes external cost from treasury leakage.
    When chain fully internalised: goods produced become free to members.
    EVER node on each printer reports:
    • What it printed (tool used, material, estimated application)
    • Application category (construction, food-tech, energy, medical, tools)
    UI: Mainsail displays local printer + network free living index progress.
